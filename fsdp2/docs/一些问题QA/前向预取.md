# 前向预取

Q:在文档<https://docs.pytorch.org/docs/2.12/distributed.fsdp.fully_shard.html>中有一句：“While fwd(m1) runs on the compute stream, the CPU fires m2’s pre-forward hook, which issues AG(b) on the AG stream. To make this overlap more robust (e.g. when CPU-side overhead reduces the lead), use set_modules_to_forward_prefetch to issue the next all-gather earlier — inside the current module’s pre-forward hook rather than waiting for the next module’s hook to fire.”。这要怎么理解？

A：翻译成：当模块 m1 的前向计算（fwd）在 GPU 的计算流（compute stream）上运行时，CPU 会触发模块 m2 的前向预钩子（pre-forward hook），该钩子会在 GPU 的通信流（AG stream）上发起 All-Gather(b) 操作。
先介绍下背景：
有一个模型，如下
model[ m1[a] -> m2[b] -> m3[c] -> m4[d] ]
写代码的时候，先使用fully_shard(m2), fully_shard(m3), 最后fully_shard(model)
那么大致的流程如下

```text
              time ──────────────────────────────────────────────►

compute:      [wait] [ fwd(m1)   | fwd(m2)    | fwd(m3,m4)     ]
AG stream:    [AG(a,d)]  [AG(b)  |    AG(c)   ]
```

我是这样理解的，fsdp2使用bottom-up的方式，先对每个子模块进行fully_shard，最后对root模块进行fully_shard。对于一个模型，正常的流程是m1、m2、m3、m4进行前向计算的时候，按照顺序进行。自然地，需要先all gather m1的参数，才能进行m2的前向计算。但是因为我们调用fully_shard的方法是先使用fully_shard(m2), fully_shard(m3), 最后fully_shard(model)。m1、m4参数在一起，所以gather的时候，会把m1、m4一起gather。
torch的代码实现：当 device 正在计算流上计算 m1 时，CPU 会提前去触发 m2 的钩子（hook），命令 device 在通信流上开始收集 m2 的参数。这样，m1 的计算和 m2 的通信就同时进行（Overlap） 了，完美隐藏了通信延迟。这是隐式预取（Implicit Prefetching），默认行为。
但是，在实际训练中 CPU 可能会忙不过来。CPU-bound 的工作负载，比如复杂的逻辑处理、小 batch size 导致 device 计算极快。
后果：当 m1 在 device 上算完了，CPU 还没来得及触发 m2 的 hook，导致 m2 的 All-Gather 通信启动晚了。这时 device 的计算流只能停下来干等通信流把 m2 的参数all gather完，产生了device 气泡（Idle time / 性能损耗）。这就是所谓的“CPU 开销减少了提前量（reduces the lead）”。

为了解决 CPU 动作慢的问题，FSDP2 提供了 set_modules_to_forward_prefetch API。
改变触发时机：
默认情况：等 m1 开始算了，CPU 再去触发 m2 的 hook 发起通信。
优化后：在 m1 刚刚准备开始算（即进入 m1 的 pre-forward hook）时，就立刻、提前把 m2（甚至 m3）的 All-Gather 通信指令下发给 device。
好处：给 CPU 留出了充足的缓冲时间。即使 CPU 后续调度慢了，通信指令也早就提前下发到 GPU 的通信流里排队了，从而保证了“计算”和“通信”能够稳稳地重叠执行。

例子：

```python
num_to_forward_prefetch = 2
for i, layer in enumerate(model.layers):
    # 提前预取后面的 1 到 2 层
    layers_to_prefetch = [model.layers[i + j] for j in range(1, num_to_forward_prefetch + 1)]
    layer.set_modules_to_forward_prefetch(layers_to_prefetch)
```

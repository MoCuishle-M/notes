# FSDP2 前向计算详解：Stream、All-Gather 与 Hook 机制

> 基于 PyTorch v2.12.0 源码分析
> 核心源码：
>
> - `_fsdp_state.py`：`_pre_forward` / `_post_forward` / state 管理
> - `_fsdp_param_group.py`：`pre_forward` / `post_forward` / `unshard` / `wait_for_unshard`
> - `_fsdp_collectives.py`：`foreach_all_gather` / `foreach_all_gather_copy_out`

## 0. 示例代码（贯穿全文的起点）

基于以下模型配置来说明前向流程：

```python
import torch
import torch.nn as nn
import torch.distributed as dist
from torch.distributed.device_mesh import init_device_mesh
from torch.distributed.fsdp import fully_shard

dist.init_process_group(backend="nccl")
local_rank = dist.get_rank()
torch.cuda.set_device(local_rank)

mesh = init_device_mesh("cuda", (4,))  # 4 GPU FSDP

class Block(nn.Module):
    def __init__(self):
        super().__init__()
        self.fc1 = nn.Linear(512, 512)
        self.fc2 = nn.Linear(512, 512)

    def forward(self, x):
        return self.fc2(torch.relu(self.fc1(x)))

class Model(nn.Module):
    def __init__(self):
        super().__init__()
        self.block0 = Block()
        self.block1 = Block()

    def forward(self, x):
        x = self.block0(x)
        x = self.block1(x)
        return x

model = Model().cuda()
fully_shard(model.block0, mesh=mesh)  # FSDP 包装 block0
fully_shard(model.block1, mesh=mesh)  # FSDP 包装 block1
fully_shard(model, mesh=mesh)         # 顶层包装

x = torch.randn(32, 512).cuda()
output = model(x)  # ← 此时的完整前向计算流程
```

## 1. 前向计算的完整调用链

当调用 `model(x)` 时，触发以下调用链：

```text
model.forward(x)                              # FSDPModule.forward (MRO 替换后)
  └─ FSDPState._pre_forward(model, args, kwargs)   # 前向预处理 hook
       ├─ _root_pre_forward(state, model)           # 根状态同步
       │    └─ all_gather_stream.wait_stream(current_stream)  # 等待计算流
       │    └─ all_gather_copy_in_stream.wait_stream(current_stream)
       │
       ├─ for each fsdp_param_group:
       │    └─ fsdp_param_group.pre_forward(module, args, kwargs)
       │         ├─ TrainingState = FORWARD
       │         ├─ unshard()                         # 触发 all-gather
       │         └─ wait_for_unshard()               # 等待 all-gather 完成
       │
       └─ _prefetch_forward: 预取下一层的 all-gather

  └─ 原始 model.forward(x) 执行                # 实际的前向计算

  └─ FSDPState._post_forward(model, args, output)  # 前向后处理 hook
       ├─ for each fsdp_param_group (逆序):
       │    └─ fsdp_param_group.post_forward()
       │         └─ reshard()                        # 释放完整参数
       └─ TrainingState = IDLE
```

## 2. _pre_forward 详解：Hook 触发与 All-Gather 流程

### 2.1 根前向预处理 (_root_pre_forward)

```python
# _fsdp_state.py
def _root_pre_forward(self, module: nn.Module):
    if self._training_state in (TrainingState.FORWARD, TrainingState.PRE_BACKWARD):
        return  # 防止重入（SAC 重计算场景）
    
    self._lazy_init(module)  # 延迟初始化（首次调用时）
    
    # 同步流：确保优化器步骤已完成
    # 场景 1: 用户提供了 post_optim_event
    if self._post_optim_event is not None:
        self._all_gather_copy_in_stream.wait_event(self._post_optim_event)
        self._all_gather_stream.wait_event(self._post_optim_event)
        self._post_optim_event = None
    # 场景 2: 默认同步
    else:
        self._device_handle.current_stream().wait_stream(
            self._all_gather_copy_in_stream
        )
        self._device_handle.current_stream().wait_stream(
            self._all_gather_stream
        )
```

**为什么需要根流同步？** FSDP2 使用独立的 CUDA stream 进行 all-gather 通信。如果上次反向的 reduce-scatter 还在进行中，新的前向必须在那些操作完成后才能开始读取参数。

### 2.2 参数组的前向预处理

```python
# _fsdp_param_group.py: FSDPParamGroup.pre_forward()
def pre_forward(self, module, args, kwargs):
    self._training_state = TrainingState.FORWARD  # 状态切换
    
    # 1. Unshard: 将分片参数 all-gather 为完整参数
    self.unshard()
    
    # 2. Wait: 等待 all-gather 操作完成
    self.wait_for_unshard()
    
    # 3. 注册 pre-backward hook（在输出张量上）
    self._register_post_backward_hook(args, kwargs)
```

### 2.3 Unshard 触发 All-Gather

```python
# _fsdp_param_group.py: FSDPParamGroup.unshard()
def unshard(self, async_op=False):
    if self.is_unsharded:
        return  # 已经是完整的，跳过
    
    if self.world_size == 1:
        self._handle = AllGatherState(None, None, None)  # 单 GPU 不需要通信
        return
    
    # 收集所有要 all-gather 的参数输入（分片数据）
    all_gather_inputs = []
    all_gather_numels = []
    for fsdp_param in self.fsdp_params:
        inputs = fsdp_param.all_gather_inputs  # 获取 1D 分片数据
        all_gather_inputs.extend(inputs)
        all_gather_numels.append(inputs[0].numel())
    
    # === 关键：双流异步通信 ===
    # 调用 foreach_all_gather（在 _fsdp_collectives.py 中）
    self._handle = foreach_all_gather(
        all_gather_inputs,
        all_gather_numels,
        self.fsdp_params,
        group=self.mesh_info.shard_process_group,
        all_gather_comm=self.all_gather_comm,
        copy_in_stream=self.comm_ctx.all_gather_copy_in_stream,
        all_gather_stream=self.comm_ctx.all_gather_stream,
        async_op=async_op,
    )
```

## 3. CPU/GPU Stream 交互详解

FSDP2 使用 **三个 CUDA stream** 实现通信-计算重叠：

```text
┌─────────────────────────────────────────────────────────────────────────┐
│                           时间线                                         │
├─────────────────────────────────────────────────────────────────────────┤
│ 计算流 (current stream)  │ 前向计算 block0 │ 前向计算 block1 │ ...        │
├─────────────────────────────────────────────────────────────────────────┤
│ all_gather_copy_in_stream │ copy-in B1  │ copy-in B2  │                │
├─────────────────────────────────────────────────────────────────────────┤
│ all_gather_stream         │   all-gather B0   │ all-gather B1  │       │
└─────────────────────────────────────────────────────────────────────────┘
```

### 3.1 Stream 优先级设置

```python
# _fsdp_param_group.py: FSDPCommContext
class FSDPCommContext:
    def __init__(self):
        device_handle = _get_device_handle(device_type)
        
        # 高优先级流 — 通信流
        self.all_gather_copy_in_stream = device_handle.Stream(priority=-1)
        self.all_gather_stream = device_handle.Stream(priority=-1)
        self.reduce_scatter_stream = device_handle.Stream(priority=-1)
        
        # 普通优先级流 — HSDP all-reduce 流
        self.all_reduce_stream = device_handle.Stream(priority=0)
```

### 3.2 All-Gather 双流异步模式

源码中 `foreach_all_gather` 的核心实现体现了**双流流水线**：

```python
# _fsdp_collectives.py: foreach_all_gather()
def foreach_all_gather(
    all_gather_inputs, all_gather_numels, fsdp_params,
    group, all_gather_comm, copy_in_stream, all_gather_stream,
    async_op=False,
):
    device_handle = _get_device_handle(group.device_type)
    
    # ====== 阶段 1: Copy-In (在 copy_in_stream 上执行) ======
    with device_handle.stream(copy_in_stream):
        # 1.1 获取所有参数的 all-gather 输入
        #     _get_param_all_gather_inputs 返回展平的 1D tensor 列表
        all_gather_inputs = _get_param_all_gather_inputs(fsdp_params)
        
        # 1.2 分配 all-gather 输出缓冲区
        #     大小为 all_gather_input_numel * world_size
        #     每个 rank 的输入将被放在缓冲区的对应分区中
        self_all_gather_input_numel = sum(inp.numel() for inp in all_gather_inputs)
        all_gather_output = all_gather_comm.allocate(
            self_all_gather_input_numel * group.size()
        )
        
        # 1.3 执行 copy-in: 将分片数据复制到输出缓冲区的对应位置
        #     rank=i 的数据被复制到 output[i*numel : (i+1)*numel]
        torch.ops.fsdp.all_gather_copy_in(
            all_gather_inputs,
            all_gather_output,
            inp_split_sizes,
            self_all_gather_input_numel,
            group.rank(),
        )
        # all_gather_copy_in 内部实现：
        #   output_narrowed = output.narrow(0, rank*numel, numel)
        #   torch._foreach_copy_(output_narrowed_chunks, all_gather_inputs)
    
    # ====== 同步: all_gather_stream 等待 copy_in_stream ======
    all_gather_stream.wait_stream(copy_in_stream)
    
    # ====== 阶段 2: All-Gather (在 all_gather_stream 上执行) ======
    with device_handle.stream(all_gather_stream):
        # 2.1 执行 all-gather 通信（NCCL 内核）
        all_gather_work = all_gather_comm(all_gather_output, group)
        # 等价于: dist.all_gather_into_tensor(all_gather_output, ..., group=group)
        
        # 2.2 记录事件（用于 copy-out 同步）
        all_gather_event = all_gather_stream.record_event()
    
    return AllGatherState(all_gather_output, all_gather_event, all_gather_work)
```

### 3.3 Copy-Out：等待 All-Gather 完成

```python
# _fsdp_collectives.py: foreach_all_gather_copy_out()
def foreach_all_gather_copy_out(
    all_gather_state, fsdp_params, group, symm_mem_comm,
):
    all_gather_output, all_gather_event, all_gather_work = all_gather_state
    
    device_handle = _get_device_handle(group.device_type)
    
    # 同步: 计算流等待 all_gather 完成
    if all_gather_event is not None:
        device_handle.current_stream().wait_event(all_gather_event)
    if all_gather_work is not None:
        all_gather_work.wait()
    
    # 将 all-gather 输出拆分到各参数的输出缓冲区
    torch.ops.fsdp.split_with_sizes_copy(
        all_gather_output, all_gather_output_splits, ...
    )
    
    # 对各参数调用 init_unsharded_param() 建立完整参数视图
    for fsdp_param in fsdp_params:
        fsdp_param.init_unsharded_param()
```

### 3.4 Stream 交互时序图

```text
时间 →

计算流 (default):
  [ pre_forward block0 ][ block0.forward() 计算 ]────[ pre_forward block1 ][ block1.forward() 计算 ]
              │                          ▲                        │                          ▲
              │ 触发                     │ 等待                   │ 触发                     │ 等待
              ▼                          │                        ▼                          │
copy_in_stream:
  [ copy_in B0 ] ────────────────────────┼── [ copy_in B1 ] ──────────────────────────────┼──
              │                          │                        │                          │
              │ wait                     │                        │ wait                     │
              ▼                          │                        ▼                          │
all_gather_stream:
  [ all-gather B0 ] ── event ────────────┘  [ all-gather B1 ] ── event ───────────────────┘
```

**关键点**：

- **copy_in 和 all_gather 在独立的高优先级流上执行**，与计算流并行
- **计算流在 `wait_for_unshard()` 时等待 all_gather event**，确保参数可用
- **预取机制**：block0 在计算时，block1 的 all-gather 已经开始

## 4. 前向预取（Forward Prefetch）机制

### 4.1 隐式预取

```python
# _fsdp_state.py: _pre_forward (简化)
for fsdp_param_group in self._fsdp_param_groups:
    fsdp_param_group.pre_forward(...)  # 当前组的 unshard + wait
    
    # --- 预取下一个模块的参数组 ---
    for target_state in self._states_to_forward_prefetch:
        for target_group in target_state._fsdp_param_groups:
            FSDPParamGroup._prefetch_unshard(target_group, "forward")
```

### 4.2 显式预取设置

```python
# 用户可以显式指定预取顺序
model.block0.set_modules_to_forward_prefetch([model.block1])
model.block1.set_modules_to_forward_prefetch([model.block2])
```

### 4.3 预取的实现

```python
# _fsdp_param_group.py
@staticmethod
def _prefetch_unshard(target_param_group, direction):
    target_param_group._training_state = TrainingState.FORWARD
    target_param_group.unshard()  # 异步触发 all-gather，不等待
```

预取只是**触发** all-gather，不等待完成。实际的等待发生在目标模块的 `pre_forward` 调用 `wait_for_unshard()` 时。

## 5. _post_forward 详解：释放完整参数

```python
# _fsdp_state.py: FSDPState._post_forward()
def _post_forward(self, module, args, output):
    if self._training_state == TrainingState.PRE_BACKWARD:
        # SAC 重计算场景：不执行 reshard
        return output
    
    for fsdp_param_group in self._fsdp_param_groups:
        fsdp_param_group.post_forward()
        # 记录到 post_forward_order（用于反向预取）
        self._comm_ctx.post_forward_order.append(fsdp_param_group)
    
    # 释放最后一个 all-gather 结果
    # 参考注释: [Note: Overlapping all-gather copy-in and all-gather]
    self._comm_ctx.all_gather_state = None
    
    self._training_state = TrainingState.IDLE
    return output
```

```python
# _fsdp_param_group.py: FSDPParamGroup.post_forward()
def post_forward(self):
    self.reshard()  # 释放完整参数，注册分片参数
    self._training_state = TrainingState.IDLE
```

### 5.1 reshard 的两种模式

```python
def reshard(self):
    if self.post_forward_mesh_info is not None:
        # HSDP 模式：reshard 到 post_forward_mesh
        self._to_sharded_post_forward()
    else:
        # FSDP 模式：reshard 回原始分片
```

## 6. Hook 机制总结

FSDP2 使用**三套 hook 机制**实现前向计算拦截：

### 6.1 MRO 替换 Hook（最外层）

```python
# _fully_shard.py: _apply_to_module()
# 通过修改 __class__ 将 FSDPModule 插入 MRO
# FSDPModule.forward 天然成为第一个被调用的 forward
```

### 6.2 register_fsdp_forward_method（自定义方法）

```python
# 如果用户自定义了 forward-like 方法（非 __call__）
register_fsdp_forward_method(module, "my_forward")
# 内部包装：
#   wrapped_func = _pre_forward -> original_method -> _post_forward
```

### 6.3 前向 hook 注册时机

```text
FSDPModule.__new__ (MRO 替换)
     └─ FSDPModule.forward 重写 __call__
          └─ _pre_forward   ← 在原始 forward 之前执行
               └─ pre_forward  (每个 FSDPParamGroup)
          └─ original_forward
          └─ _post_forward  ← 在原始 forward 之后执行
               └─ post_forward (每个 FSDPParamGroup)
```

### 6.4 与前向计算流程的完整对应

| 阶段 | Hook | 位置 | 操作 |
| ------ | ------ | ------ | ------ |
| pre-forward | `_root_pre_forward` | `_fsdp_state.py` | Stream 同步、延迟初始化 |
| pre-forward | `pre_forward` | `_fsdp_param_group.py` | 触发 all-gather、等待完成 |
| pre-forward | `_prefetch_unshard` | `_fsdp_param_group.py` | 预取下一层参数 |
| compute | `module.forward()` | 用户代码 | 实际前向计算 |
| post-forward | `post_forward` | `_fsdp_param_group.py` | Reshard 释放参数 |
| post-forward | `_post_forward` | `_fsdp_state.py` | 清理 all-gather 状态 |

## 1. 概念：什么是 contiguous

### 1.1 定义

一个 tensor 的数据在内存中按 **逻辑维度顺序（row-major，最后一维连续）** 排列时，称为 contiguous。

对于形状 `(d0, d1, ..., dn)` 的标准 contiguous tensor，其 stride 满足：

```
stride[i] = stride[i+1] * size[i+1]   (最后一维 stride == 1)
```

例如 `torch.randn(2, 3, 4)` 的 stride 为 `(12, 4, 1)`。

### 1.2 判断方法

```python
tensor.is_contiguous()                       # 默认按 contiguous_format 判断
tensor.is_contiguous(memory_format=torch.channels_last)  # 按 channels_last 判断
```

**关键点**：`is_contiguous()` 默认是 `contiguous_format`（row-major）。
**channels_last tensor 虽然在它自己的格式下是"连续"的，但 `is_contiguous()` 返回 `False`**。这正是 FSDP 拒绝它的原因。

### 1.3 stride 的本质

PyTorch tensor = 一段扁平 `storage` + `offset` + `shape` + `stride`。
凡是**只改 stride / shape / offset、不搬动数据**的操作，都可能产生 non-contiguous tensor。

## 2. FSDP2 的拒绝逻辑

`_fsdp_param.py` 第 262-264 行（与 [分片边界条件与dtype要求.md](../分片边界条件与dtype要求.md) §4.5 一致）：

```python
if not param.is_contiguous():
    raise NotImplementedError(
        f"FSDP does not support non-contiguous parameters yet: {param.shape=} {param.stride()=}"
    )
```

### 2.1 为什么 FSDP 要求连续

FSDP 的 all-gather / reduce-scatter 集体通信在参数的 **flat 一维视图** 上操作：

- `_fsdp_collectives.py` 中 `foreach_all_gather` / `foreach_reduce_scatter` 直接用 `param._flat_view` 或按 dim 0 切分；
- non-contiguous 参数的 stride 不规则，无法直接当作连续 buffer 参与 NCCL 通信，会引入额外 copy 和 padding 逻辑（与 uneven sharding 类似）；
- 当前实现选择**直接拒绝**而非内部 `.contiguous()`，是为了避免隐式拷贝带来的性能惊吓和语义歧义。

测试用例 `test_raise_noncontiguous_parameter`（`test_fully_shard_init.py` 第 453-460 行）：

```python
def test_raise_noncontiguous_parameter(self):
    conv2d = nn.Conv2d(8, 8, 3).to(memory_format=torch.channels_last)
    with self.assertRaisesRegex(
        NotImplementedError, "FSDP does not support non-contiguous parameters"
    ):
        fully_shard(conv2d)
```

## 3. 产生 non-contiguous tensor 的常见操作

> 下列操作都返回 **view**（共享 storage），不会拷贝数据，因此会把"非连续性"传递下去。
> 对应示例：`示例代码/02_*.py` ~ `示例代码/04_*.py`

### 3.1 memory format 转换（FSDP 测试用例就是这一类）

| 操作 | 是否 non-contiguous（默认判断） | 说明 |
| --- | --- | --- |
| `t.to(memory_format=torch.channels_last)` | ✅ 是 | 4D tensor 的 stride 变成 `(C*H*W, 1, C*W, C)` |
| `t.to(memory_format=torch.channels_last_3d)` | ✅ 是 | 5D tensor 的 channels_last 变体 |
| `t.to(memory_format=torch.preserve_format)` | 保持原状 | 不改变 contiguity |

```python
conv = nn.Conv2d(8, 8, 3)
conv = conv.to(memory_format=torch.channels_last)
print(conv.weight.is_contiguous())           # False  ← FSDP 拒绝
print(conv.weight.is_contiguous(torch.channels_last))  # True
```

### 3.2 `transpose` / `permute` / `T` / `t()`

```python
x = torch.randn(3, 4)            # contiguous
y = x.T                          # shape (4,3), stride (1,4)  → non-contiguous
z = x.transpose(0, 1)            # 同上
w = x.permute(1, 0)              # 同上
```

> 高维 permute 一定 non-contiguous（除非 permute 后顺序恰好等于原顺序）。

### 3.3 strided slicing / `narrow` / `index_select`

```python
x = torch.randn(6, 8)
x[:, ::2]                        # 步长切片 → non-contiguous
x[:, 1:]                         # 在非第 0 维切 → non-contiguous
x.narrow(1, 1, 4)                # 等价于 x[:, 1:5] → non-contiguous
x[1:, :]                         # 在第 0 维切 → 仍然 contiguous
```

**判断口诀**：只有**在第 0 维上做起点偏移**的切片保持 contiguous；任何改变 stride（步长）或在非 0 维做范围切片，都会变 non-contiguous。

### 3.4 `expand` / `expand_as`

```python
x = torch.randn(1, 5)            # contiguous
y = x.expand(3, 5)               # shape (3,5), stride (0,1) → non-contiguous
```

> 引入 stride=0 的维度后，`is_contiguous()` 返回 `False`。

### 3.5 `flip` / `fliplr` / `flipud` —— **返回 contiguous copy，不是非连续**

> 注意：直觉上 `flip` 应该返回带负 stride 的 view，但 **PyTorch 2.12 的 `torch.flip` 实际返回 contiguous 拷贝**（见 `示例代码/04_expand_flip_as_strided.py` 实测：翻转后 `stride=(4,1)`，`is_contiguous()=True`）。
> 因此 `flip` **不会** 产生 non-contiguous 参数，不会触发 FSDP 的拒绝。下文不再把它列为风险源。

### 3.6 `as_strided`

```python
storage = torch.randn(12)
y = torch.as_strided(storage, size=(3, 4), stride=(4, 1))  # 可任意指定 → 通常 non-contiguous
```

### 3.7 `diagonal` —— 非连续；`triu` / `tril` —— contiguous copy

```python
x = torch.randn(4, 4)
y = torch.diagonal(x, offset=1)  # 跨步访问 stride=(5,) → non-contiguous
z = torch.triu(x, diagonal=1)    # 实测返回 contiguous copy（stride=(4,1)）
w = torch.tril(x, diagonal=-1)   # 同上，contiguous copy
```

> 实测结论（PyTorch 2.12，见 `示例代码/04_*.py`）：`diagonal` 返回非连续 view；`triu` / `tril` 返回连续拷贝。只有 `diagonal` 是风险源。

## 4. 产生 non-contiguous `nn.Parameter` 的实际场景

> `nn.Parameter(data)` 内部通过 `Tensor._make_subclass` 构造，**不会自动 `.contiguous()`**，
> 所以传入的 tensor 是什么样，参数就是什么样。
> 对应示例：`示例代码/05_*.py`

### 4.1 channels_last 卷积模型（最常见，也是 FSDP 测试用例）

```python
model = nn.Conv2d(8, 8, 3).to(memory_format=torch.channels_last)
# model.weight.is_contiguous() == False  → fully_shard(model) 直接抛错
```

> 这是 FSDP2 官方测试唯一覆盖的场景，也是用户最容易踩到的坑。

### 4.2 手动赋值 transposed / permuted tensor 作为参数

```python
linear = nn.Linear(10, 5)
# 想把权重转置存（某些 LoRA / tying 实现）
linear.weight = nn.Parameter(linear.weight.T.contiguous())  # OK
linear.weight = nn.Parameter(linear.weight.T)               # ❌ non-contiguous
```

### 4.3 从大参数切片得到子参数

```python
big = nn.Parameter(torch.randn(10, 20))
child = nn.Linear(5, 20)
child.weight = nn.Parameter(big[:5, :])    # 第 0 维切片 → 仍然 contiguous
child.weight = nn.Parameter(big[:, :10])   # 非 0 维切片 → non-contiguous
```

### 4.4 expand 用于 tied / broadcast 权重

```python
weight = nn.Parameter(torch.randn(1, 8, 4))
# 想让 3 层共享同一份权重（view 共享 storage）
tied = nn.Parameter(weight.expand(3, 8, 4))   # ❌ stride 含 0 → non-contiguous
```

### 4.5 自定义初始化 / 预处理

任何在 `__init__` 中通过 `transpose` / `flip` / `as_strided` / `diagonal` 构造权重，并直接包成 `nn.Parameter` 的写法，都有风险。

## 5. 修复方法

### 5.1 通用：`.contiguous()` 拷贝成 row-major

```python
param = nn.Parameter(non_contiguous_tensor.contiguous())
```

代价：一次拷贝、多一份显存。但这是 FSDP 推荐路径。

### 5.2 channels_last 的特别说明

- channels_last 本身是**性能优化**（对 Conv2d 在 Tensor Core 上更快）；
- 但 FSDP2 目前不支持，所以**进入 FSDP 前必须退回 contiguous**：

```python
conv = nn.Conv2d(8, 8, 3).to(memory_format=torch.channels_last)
conv = conv.to(memory_format=torch.contiguous_format)   # 退回 row-major
fully_shard(conv)                                       # OK
```

- 若一定要在 forward 中享受 channels_last，可在 `forward()` 内部临时 `.to(memory_format=torch.channels_last)`，参数本身保持 contiguous。

### 5.3 重新构造而非 view

对 §4.3 / §4.4 的场景，用 `clone()` / 重新 `torch.randn` 代替切片 / expand，从源头避免 view：

```python
child.weight = nn.Parameter(big[:5, :].contiguous().clone())
```

### 5.4 验证脚本

`fully_shard` 之前做一次体检：

```python
def check_contiguous(module: nn.Module):
    bad = []
    for name, p in module.named_parameters():
        if not p.is_contiguous():
            bad.append((name, tuple(p.shape), tuple(p.stride())))
    assert not bad, f"non-contiguous params: {bad}"
```

## 7. 结论速查表

| 场景 | 是否产生 non-contiguous | FSDP 是否接受 |
| --- | --- | --- |
| `nn.Linear` 默认权重 | 不产生 | 接受 |
| `nn.Conv2d` + `channels_last` | 产生 | 拒绝 |
| `param.T` 直接当参数 | 产生 | 拒绝 |
| `param[:, :n]` 切片当参数（非 0 维） | 产生 | 拒绝 |
| `param[:n, :]` 切片当参数（第 0 维） | 不产生 | 接受 |
| `param.expand(...)` 当参数 | 产生 | 拒绝 |
| `torch.flip(param, ...)` 当参数 | 不产生（2.12 返回 contiguous copy） | 接受 |
| `torch.diagonal(...)` 当参数 | 产生 | 拒绝 |
| `torch.triu` / `tril` 当参数 | 不产生（2.12 返回 contiguous copy） | 接受 |
| `torch.as_strided(...)` 当参数 | 产生 | 拒绝 |
| 上述任意 + `.contiguous()` | 不产生 | 接受 |

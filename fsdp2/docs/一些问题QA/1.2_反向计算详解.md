# FSDP2 反向计算详解：Stream、Reduce-Scatter 与 Hook 机制

> 基于 PyTorch v2.12.0 源码分析
> 核心源码：
>
> - `_fsdp_state.py`：`_pre_backward` / `_root_post_backward_final_callback` / `TrainingState`
> - `_fsdp_param_group.py`：`pre_backward` / `post_backward` / `finalize_backward`
> - `_fsdp_collectives.py`：`foreach_reduce`（reduce-scatter + all-reduce）

## 0. 示例代码（贯穿全文的起点）

```python
import torch
import torch.nn as nn
import torch.distributed as dist
from torch.distributed.device_mesh import init_device_mesh
from torch.distributed.fsdp import fully_shard

dist.init_process_group(backend="nccl")
local_rank = dist.get_rank()
torch.cuda.set_device(local_rank)

mesh = init_device_mesh("cuda", (4,))  # 4 GPU FSDP

class Block(nn.Module):
    def __init__(self):
        super().__init__()
        self.fc1 = nn.Linear(512, 512)
        self.fc2 = nn.Linear(512, 512)

    def forward(self, x):
        return self.fc2(torch.relu(self.fc1(x)))

class Model(nn.Module):
    def __init__(self):
        super().__init__()
        self.block0 = Block()
        self.block1 = Block()

    def forward(self, x):
        x = self.block0(x)
        x = self.block1(x)
        return x

model = Model().cuda()
fully_shard(model.block0, mesh=mesh)
fully_shard(model.block1, mesh=mesh)
fully_shard(model, mesh=mesh)

x = torch.randn(32, 512).cuda()
output = model(x)
loss = output.sum()
loss.backward()  # ← 此时的完整反向计算流程
```

## 1. 反向计算的完整调用链

当调用 `loss.backward()` 时，触发以下调用链：

```text
loss.backward()
  │
  ├─ Autograd 引擎按反向拓扑序执行梯度计算
  │
  ├─ 当梯度到达 model.lm_head.weight 时:
  │    └─ 注册在输出上的 pre-backward hook 被触发:
  │
  ├─ FSDPState._pre_backward(module)   ← 由前向时注册的 hook 触发
  │    ├─ root_state._training_state = PRE_BACKWARD
  │    └─ 对每个 fsdp_param_group (逆序):
  │         ├─ pre_backward():
  │         │    └─ unshard() + wait_for_unshard()  ← 为反向计算恢复完整参数
  │         └─ _prefetch_unshard(target_group, "backward") ← 预取上一层
  │
  ├─ Autograd 引擎继续反向计算（使用完整参数计算梯度）
  │
  └─ Autograd 引擎完成所有梯度计算后:
       └─ _root_post_backward_final_callback()
            └─ 对每个 fsdp_param_group (逆序):
                 └─ post_backward():
                      ├─ 收集 unsharded_grad_data
                      ├─ reshard() (如果 reshard_after_backward)
                      └─ foreach_reduce():
                           ├─ chunk_cat (copy-in)
                           ├─ reduce_scatter_tensor
                           ├─ all_reduce (HSDP 时)
                           └─ 写入 sharded gradients
```

## 2. Pre-Backward Hook 注册机制

在前向计算的 `post_forward` 阶段，FSDP2 在输出张量上注册了 pre-backward hook：

```python
# _fsdp_param_group.py: FSDPParamGroup.pre_forward()
def pre_forward(self, module, args, kwargs):
    self._training_state = TrainingState.FORWARD
    self.unshard()
    self.wait_for_unshard()
    # 关键：注册 pre-backward hook
    self._register_post_backward_hook(args, kwargs)

def _register_post_backward_hook(self, args, kwargs):
    # 在输入张量上注册 hook（确保反向传播时触发）
    for tensor in _get_grad_tensors(args, kwargs):
        tensor.register_post_accumulate_grad_hook(
            self._post_accumulate_grad_hook
        )
```

**注意**：PyTorch 的 `register_post_accumulate_grad_hook` 在梯度累加完成后触发（即梯度已计算但尚未 reduce），这正是 FSDP2 需要的时机。

## 3. _pre_backward 详解：为反向恢复完整参数

```python
# _fsdp_state.py
def _pre_backward(self, module):
    # 1. 设置训练状态（与 SAC 重计算协作）
    self._training_state = TrainingState.PRE_BACKWARD
    
    # 2. 对每个参数组执行 pre_backward
    for fsdp_param_group in self._fsdp_param_groups:
        fsdp_param_group.pre_backward(default_prefetch=True)
```

### 3.1 参数组的 pre_backward

```python
# _fsdp_param_group.py: FSDPParamGroup.pre_backward()
def pre_backward(self, default_prefetch=True):
    # 防止重入（多个梯度 hook 可能触发同一个参数组）
    if self._training_state == TrainingState.PRE_BACKWARD:
        return
    self._training_state = TrainingState.PRE_BACKWARD
    
    # 如果参数尚未 unshard（尚未通过预取恢复）
    if not self.is_unsharded:
        self.unshard()           # 异步触发 all-gather
        self.wait_for_unshard()  # 等待完成
    
    # 预取上一个模块的参数组（反向执行顺序的"上一个"）
    if default_prefetch:
        self._backward_prefetch()
```

### 3.2 反向预取机制

```python
# _fsdp_param_group.py
def _backward_prefetch(self):
    # 从 post_forward_order 中查找当前组之前的组
    # 注意：反向执行顺序 == 前向执行顺序的逆序
    # 如果当前是 block1（前向执行 block0→block1，反向执行 block1→block0）
    # 那么需要预取 block0
    comm_ctx = self.comm_ctx
    post_forward_order = comm_ctx.post_forward_order
    
    # 找到当前组在 post_forward_order 中的位置
    # 预取前一个组的参数
    curr_idx = post_forward_order.index(self)
    for i in range(curr_idx - 1, -1, -1):
        target_group = post_forward_order[i]
        if target_group._training_state != TrainingState.POST_BACKWARD:
            FSDPParamGroup._prefetch_unshard(target_group, "backward")
            break
```

**反向预取的核心思想**：block1 在做反向计算时，block0 的 all-gather 已经开始，实现通信-计算重叠。

## 4. Post-Backward 详解：梯度 Reduce-Scatter

当当前模块的梯度计算完成后（通过 `_post_accumulate_grad_hook` 检测），`post_backward` 被调用：

```python
# _fsdp_param_group.py: FSDPParamGroup.post_backward()
def post_backward(self):
    if self._training_state == TrainingState.POST_BACKWARD:
        return  # 幂等性保证
    self._training_state = TrainingState.POST_BACKWARD
    
    # 1. 收集非分片梯度数据
    unsharded_grads = []
    for fsdp_param in self.fsdp_params:
        if fsdp_param.unsharded_accumulated_grad is not None:
            unsharded_grads.append(fsdp_param.unsharded_accumulated_grad_data)
        else:
            unsharded_grads.append(fsdp_param.unsharded_grad_data)
    
    # 2. Reshard 参数（如果不需要在反向保留完整参数）
    if self.reshard_after_backward:
        self.reshard()
    
    # 3. 执行 reduce-scatter（核心通信操作）
    if self.reduce_grads:
        self._foreach_reduce_impl(unsharded_grads)
```

## 5. Reduce-Scatter 多流管线详解

`foreach_reduce` 是反向计算中最复杂的通信操作，涉及多条 stream 的协作：

```python
# _fsdp_collectives.py: foreach_reduce()
def foreach_reduce(
    unsharded_grads,       # 非分片梯度列表
    fsdp_params,           # FSDP 参数列表
    reduce_scatter_comm,   # reduce-scatter 通信器
    all_reduce_comm,       # all-reduce 通信器（HSDP）
    ...
):
    device_handle = _get_device_handle(device_type)
    world_size = group.size()
    
    # ====== Stream 设置 ======
    # 使用独立的 stream 实现异步通信
    reduce_scatter_stream = comm_ctx.reduce_scatter_stream
    all_reduce_stream = comm_ctx.all_reduce_stream
    post_reduce_stream = comm_ctx.post_reduce_stream
    
    # ====== 阶段 1: Copy-In (在计算流上执行) ======
    # 1.1 梯度类型统一（mixed precision）
    # 1.2 计算 divide factor（pre/post-divide 策略）
    predivide_factor, postdivide_factor = _get_gradient_divide_factors(...)
    
    # 1.3 Chunk-Cat: 将梯度重组为按 world_size 分块的形式
    reduce_scatter_input = torch.empty(...)
    torch.ops.fsdp.chunk_cat(
        unsharded_grads, dim=0,
        num_chunks=world_size,
        out=reduce_scatter_input
    )
    # chunk_cat 将每个梯度沿 dim=0 分成 world_size 份，交错拼接：
    # input = [grad0_chunk0, grad1_chunk0, ..., grad0_chunk1, grad1_chunk1, ...]
    
    # 同步: reduce_scatter_stream 等待当前流（copy-in 完成）
    reduce_scatter_stream.wait_stream(device_handle.current_stream())
    
    # 释放非分片梯度引用（减少内存）
    del unsharded_grads  # ← 关键：尽早释放内存
    
    # ====== 阶段 2: Reduce-Scatter (在 reduce_scatter_stream 上执行) ======
    with device_handle.stream(reduce_scatter_stream):
        # 2.1 分配 reduce 输出缓冲区
        reduce_output = reduce_scatter_comm.allocate(
            reduce_scatter_input_numel // world_size
        )
        
        # 2.2 预除（如果使用 ReduceOp.SUM 策略）
        if predivide_factor != 1.0:
            reduce_scatter_input.div_(predivide_factor)
        
        # 2.3 执行 reduce-scatter
        if world_size > 1:
            reduce_scatter_work = reduce_scatter_comm(
                reduce_output, reduce_scatter_input, group=group
            )
            # 等价于: dist.reduce_scatter_tensor(reduce_output, reduce_scatter_input, ...)
        else:
            reduce_output.copy_(reduce_scatter_input / gradient_divide_factor)
        
        # 2.4 记录事件
        reduce_scatter_event = reduce_scatter_stream.record_event()
    
    # ====== 阶段 3: All-Reduce (HSDP 场景, 在 all_reduce_stream 上) ======
    if all_reduce_group is not None:
        with device_handle.stream(all_reduce_stream):
            # 3.1 等待 reduce-scatter 完成
            all_reduce_stream.wait_stream(reduce_scatter_stream)
            
            # 3.2 执行 all-reduce
            dist.all_reduce(reduce_output, group=all_reduce_group, op=all_reduce_op)
            
            all_reduce_event = all_reduce_stream.record_event()
    
    # ====== 阶段 4: Post-Reduce 处理 (在 post_reduce_stream 上) ======
    with device_handle.stream(post_reduce_stream):
        # 4.1 等待 reduce-scatter（和 all-reduce）
        post_reduce_stream.wait_stream(reduce_scatter_stream)
        if all_reduce_group is not None:
            post_reduce_stream.wait_stream(all_reduce_stream)
        
        # 4.2 后除（数值稳定性）
        if postdivide_factor != 1.0:
            reduce_output.div_(postdivide_factor)
        
        # 4.3 类型转换（如果使用了 mixed precision）
        if reduce_dtype != orig_dtype:
            reduce_output = reduce_output.to(orig_dtype)
        
        # 4.4 写入分片梯度（as_strided 视图）
        for i, fsdp_param in enumerate(fsdp_params):
            sharded_grad = torch.as_strided(reduce_output, ...)
            fsdp_param.sharded_param.grad = sharded_grad
        
        # 4.5 CPU offload（如果需要）
        if offload_to_cpu:
            sharded_grad = sharded_grad.to("cpu", non_blocking=True)
        
        # 4.6 累加梯度（梯度累积场景）
        if accumulate_grads:
            fsdp_param.accumulated_sharded_grad.copy_(sharded_grad)
    
    return foreach_reduce_result
```

## 6. Copy-Out（反向的 Copy-In/Copy-Out）

在反向计算中，"copy-in" 和 "copy-out" 的角色与前向相反：

| 阶段 | 操作 | 说明 |
| ------ | ------ | ------ |
| Copy-In | `chunk_cat` | 将非分片梯度重组为 reduce-scatter 输入格式 |
| Copy-Out | `as_strided` + `copy_` | 将 reduce-scatter 输出写入各参数的分片梯度 |

### 6.1 Chunk-Cat 详解

```python
# torch.ops.fsdp.chunk_cat 的操作
# 假设 4 GPU, 2 个参数，梯度形状均为 (512, 512)

# 输入: [grad_fc1, grad_fc2]  每个形状 (512, 512)
# 输出: reduce_scatter_input 形状 (512*2, 512)

# chunk_cat 沿 dim=0 将每个梯度分成 4 份（world_size=4）
# 输出布局:
# [grad_fc1_chunk0(128,512), grad_fc2_chunk0(128,512),   ← rank0 负责
#  grad_fc1_chunk1(128,512), grad_fc2_chunk1(128,512),   ← rank1 负责
#  grad_fc1_chunk2(128,512), grad_fc2_chunk2(128,512),   ← rank2 负责
#  grad_fc1_chunk3(128,512), grad_fc2_chunk3(128,512)]   ← rank3 负责
```

## 7. Stream 交互时序图

```text
时间 →

计算流 (default):
  [block1 反向计算]──[block0 反向计算]──[ ... ]──[ 回调: post_backward ]
         │                   │                              │
         │ 触发预取           │ 触发预取                       │ 触发 chunk_cat
         ▼                   ▼                              ▼
all_gather_copy_in_stream:
  [copy_in B0] ──────────────────────────────────────────   (预取上一层的 all-gather)
         │
         ▼
all_gather_stream:
  [all-gather B0]──event──                                 (预取通信)
                              │
reduce_scatter_stream:        │                              ┌[wait]──[reduce-scatter B1]──event
                              │                              │
                              └──────────────────────────────┘

all_reduce_stream (HSDP):
                                                    ┌[wait]──[all-reduce]──event

post_reduce_stream:
                                                              ┌[wait]──[后处理]
```

## 8. 根反向最终回调

`_root_post_backward_final_callback` 是反向传播结束时的最终清理：

```python
# _fsdp_state.py
def _root_post_backward_final_callback(self):
    # 1. 按逆序遍历所有 FSDP 状态（确保最外层最后处理）
    fsdp_states = _get_all_fsdp_states(self._module)
    for state in reversed(fsdp_states):
        for fsdp_param_group in reversed(state._fsdp_param_groups):
            # 2. 对尚未 POST_BACKWARD 的参数组执行 post_backward
            if fsdp_param_group._training_state != TrainingState.POST_BACKWARD:
                fsdp_param_group.post_backward()
            
            # 3. 最终化
            fsdp_param_group.finalize_backward()
        
        # 4. 重置状态
        state._training_state = TrainingState.IDLE
        state._comm_ctx.post_forward_order.clear()
    
    # 5. 等待所有 reduce-scatter 事件
    for rs_state in self._comm_ctx.reduce_scatter_states:
        if rs_state.event is not None:
            self._device_handle.current_stream().wait_event(rs_state.event)
    self._comm_ctx.reduce_scatter_states.clear()
```

## 9. 梯度除法策略

```python
# _fsdp_collectives.py: _get_gradient_divide_factors()

# 策略选择:
# 1. fp32/bf16 + 无溢出风险:
#    → ReduceOp.AVG (NCCL 内置除法)
#    → predivide_factor = postdivide_factor = 1.0
#
# 2. fp16 等溢出敏感类型:
#    → ReduceOp.SUM + 显式除法
#    → predivide_factor *= 2 (循环倍增直到 factor/pret
#      factor > pre_factor
#    → postdivide_factor = factor / predivide_factor
#    例如 factor=8: pre=2, post=4
#    
# 3. force_sum_reduction_for_comms=True (MTIA 等后端):
#    → ReduceOp.SUM + 显式除法
```

## 10. Hook 机制总结

### 反向 Hooks 链

```text
_pre_backward (module-level hook)
  └─ _pre_backward (FSDPState)
       └─ pre_backward (FSDPParamGroup)
            ├─ unshard + wait_for_unshard
            └─ _prefetch_unshard (反向预取)

_post_accumulate_grad_hook (tensor-level hook)
  └─ _post_accumulate_grad_hook
       └─ 累加梯度 / 触发 post_backward

post_backward (FSDPParamGroup)
  ├─ 收集 unsharded_grads
  ├─ reshard (可选)
  └─ foreach_reduce (reduce-scatter + all-reduce)

_root_post_backward_final_callback
  ├─ 确保所有 param_group 完成 post_backward
  ├─ 等待所有 reduce-scatter events
  └─ 清理状态，回到 IDLE
```

### Hook 注册位置

| Hook | 注册位置 | 触发时机 |
| ------ | --------- | --------- |
| `_pre_backward` | `_fsdp_state.py` 中通过 module hook 注册 | 梯度开始流入模块时 |
| `_post_accumulate_grad_hook` | `pre_forward` 时注册在输入张量上 | 梯度累加完成后 |
| `_root_post_backward_final_callback` | 根模块 lazy_init 时注册 | Autograd 引擎完成所有操作后 |
